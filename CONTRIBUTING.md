# Contributing to Gaianexus

Bienvenue! Gaianexus est un projet open source collaboratif où chaque contribution est précieuse.

## Nos principes

- **Respect mutuel** : Toutes les voix sont écoutées
- **Coopération** : Nous construisons ensemble
- **Transparence** : Les décisions sont discutées publiquement
- **Inclusivité** : Aucune discrimination

## Comment contribuer

### Signaler un bug

1. Ouvrez une **Issue** sur GitHub
2. Décrivez le problème clairement
3. Incluez des étapes pour reproduire le bug
4. Proposez une solution si vous en avez une

### Proposer une amélioration

1. Ouvrez une **Discussion** ou une **Issue**
2. Explicitez votre idée et ses bénéfices
3. Attendez les retours de la communauté
4. Collaborez pour raffiner la proposition

### Développer une fonctionnalité

1. Créez une **branche** dédiée (`feature/ma-fonctionnalite`)
2. Respectez les conventions de code du projet
3. Documentez votre code
4. Ouvrez une **Pull Request** avec une description claire
5. Participez aux discussions et ajustements

### Améliorer la documentation

La documentation est aussi importante que le code!

- Améliorez les fichiers existants
- Créez des guides et tutoriels
- Traduisez la documentation
- Signalez les erreurs ou imprécisions

## Guide de style

### Code

- Utilisez **TypeScript** pour le frontend
- Utilisez **Node.js/Express** ou **Python/FastAPI** pour le backend
- Respectez les conventions existantes
- Commentez votre code
- Écrivez des tests unitaires

### Documentation

- Utilisez **Markdown**
- Soyez clair et concis
- Incluez des exemples
- Linkez vers d'autres documents pertinents

### Commits

- Messages clairs et descriptifs en français ou anglais
- Une ligne résumée (50 caractères max)
- Détails supplémentaires si nécessaire
- Référencez les issues liées (#123)

## Processus de Pull Request

1. **Fork** le repository
2. **Créez une branche** : `git checkout -b feature/votre-feature`
3. **Committez vos changements** : `git commit -m "Votre message"`
4. **Poussez** : `git push origin feature/votre-feature`
5. **Ouvrez une PR** sur GitHub

Vérifiez que :
- ✅ Votre code respecte le style du projet
- ✅ Vous avez écrit des tests
- ✅ La documentation est à jour
- ✅ Votre commit message est clair

## Accord de contribution

En contribuant à Gaianexus, vous acceptez que votre contribution soit publiée sous la même licence que le projet.
